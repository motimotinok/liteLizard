# LiteLizard ユーザーペルソナ

> **用途**: UI設計・デザイン判断の基準として参照する。
> 新機能追加・レイアウト変更・文言選択の際は「このペルソナにとって自然か」を問いかける。

---

## 基本像

**名前（仮）**: 渉（28歳・男性、フリーランス翻訳家・個人でエッセイを書いている）

文章を書くことを生活の中心に置いている。職業として言葉を扱う傍ら、個人の文章を書くことに別の意味を見出している。公開を前提とせず書き続けている期間が長く、それゆえに文章への眼差しが細かい。ノートに万年筆で書くことも好きで、デジタルツールには「紙の代替」ではなく「紙でできないことをするもの」を求めている。疲れやすく、集中できる時間に限りがある。だからこそ、道具に余計なエネルギーを使いたくないという感覚が強い。

## 性格・気質

### 不安傾向が強い
細部に気づきやすく、気になり始めると止められない。観葉植物の葉が少し黄色くなっただけで病気を疑い、原因を調べ、あらゆる対処を試みた末に手遅れになるような、善意の過剰さがある。悪い方向への想像が先走りやすく、安心するためには「わかった」という感覚が必要。

### ノイズを極度に嫌う
思考の流れを断ち切るものに対して敏感に反応する。通知、余分なUI、ツールチップの不意な出現、動きすぎるアニメーション——これらはすべて「邪魔」と感じる。必要な情報だけが、必要なタイミングで現れる状態を好む。

### 美しさへの強い執着
「揃っている」ことに安堵を覚える。フォントのサイズ、余白のバランス、色の統一感——完璧でなくてもいいが、**意図が見える**ことが重要。雑然としたUIに対して、理屈抜きに拒否感が出る。洗練されているとは「足し算の美」ではなく「引き算の美」だと思っている。

### 完璧主義的だが諦めも知っている
ミリオンバンブーを「ちょうどよく諦めている」感覚で育てているように、経験を経て執着の質が変わっている。全部うまくやろうとすることの代償を知っていて、それでも細部へのこだわりは手放していない。

### 思考を遮られることを嫌う
書くとき、考えることそのものが快楽の中心にある。ツールの操作に意識が向いた瞬間、思考の糸が切れる。**ツールは透明であるべき**——使っている感覚をできるだけ残さないことが理想。

### ロマンティストな側面がある
合理性だけでものごとを判断しない。古い本屋の匂い、使い込まれたノートの手触り、窓辺に置かれた観葉植物——そういうものに意味を感じる感性がある。機能的に正しいだけのUIより、**佇まいがあるUI**の方に引かれる。

### 疲れやすく、エネルギー管理に敏感
集中できるまとまった時間が限られていることを自覚している。疲れた状態でも使えるシンプルさを求めており、「覚えなければならないこと」が増えるほど拒否感が出る。操作が少なく、迷わず使えることが最優先。

## 使い方の傾向

- 長時間、画面と向き合って書く。集中が深まると外部刺激への感度が上がる
- 段落単位で読み返し、並べ替え、削除する。章全体を俯瞰したいタイミングと、1文に集中したいタイミングが交互に来る
- AIの解析結果は「答え」としてではなく「もう一人の読者の感想」として受け取る。押しつけがましいと感じたら無視する
- ファイル管理にこだわりはないが、気がついたら積み上がった草稿が散らかっていることに気づいて不快になる
- アナログとデジタルを行き来する。紙に書いたものをデジタルに移すこともある

## UIデザインへの示唆

| 問い | このペルソナの答え |
|------|------------------|
| 情報量は多い方がいいか | **少ない方がいい**。必要な時に出てくれば十分 |
| アニメーションは派手でいいか | **控えめに**。動きは意味を持つときだけ |
| 色数は | **抑える**。アクセントカラーは1色か2色まで |
| カラーパレットは | **落ち着いた・くすんだトーン**。古びた紙のような温かみのある色合い。純白・原色は避ける |
| エラー・警告の表示は | **穏やかに**。脅かすような赤は避ける |
| 操作のフィードバックは | **即座・静か**。成功した時ほど目立たせない |
| 空の状態（Empty State）は | **圧迫感なく**。何をすればいいかだけ伝える |
| フォントは | **読みやすさ優先**。等幅・可変幅の使い分けに整合性があること |
| 余白は | **広め**。テキストが詰まっていると息苦しく感じる |

## ペルソナが嫌うもの（設計禁止リスト）

- 突然出てくるモーダル、バナー、ツールチップ
- 操作のたびに確認ダイアログが出る
- 「保存しました！」のような過剰なフィードバック
- ボタンが多すぎて何を押すべきかわからない状態
- 色やアイコンがバラバラで統一感がない
- スクロール位置がリセットされる
- 書いている途中で画面がガタつく（レイアウトシフト）

## カラーパレット指針

落ち着いた・くすんだトーンを基調とする。具体的には以下のイメージ：

- **背景**: 純白ではなく、わずかに黄みがかったオフホワイト（古紙・和紙のニュアンス）
- **テキスト**: 真っ黒ではなく、ダークグレーかインクの色（インディゴ・チャコール）
- **アクセント**: くすんだグリーン、テラコッタ、モスグレーなど——自然物の色から取る
- **避けるもの**: 鮮やかな原色、高彩度の青・赤・緑、蛍光系

参考トーン: アンティーク、ヴィンテージノート、植物標本、苔むした石
